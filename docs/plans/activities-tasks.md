# activities-tasks — design note (T0)

## Task entity (config-less system entity, teams/webhooks-style)
- Shape: `{ id: "tk_<n>", title, status: "todo"|"doing"|"done", due: "yyyy-mm-dd"|null, assignee: string|null (users-directory NAME, lenient like `user` fields — not hard-validated), links: [{obj, id}] (0..n), createdBy: email|null, createdAt, doneAt: ISO|null }`.
- Store ops (`server/store.mjs`, additive): `taskAdd` · `taskUpdate` (status→done stamps `doneAt` via `_now()`; leaving done clears it) · `taskRemove` (hard splice, webhook-style — tasks aren't records, no trash). All three join `LOGGED_OPS`; ids from `++this.n`, time via `_now()` — replay-deterministic.
- Timeline coupling INSIDE the ops (so replay reproduces events): `taskAdd` fires `_ev(obj, id, "updated", "Task added: <title>")` per initial link; `taskUpdate` transitioning to done fires `"Task done: <title>"` per current link. Kind is `"updated"` because `TimelineEvent.kind` is a CLOSED union in frozen `src/ui/record-core/types.ts` (a real "task" kind = nexus-ui change; not needed — pencil icon renders today). Unlink/relink stamp nothing (kept to brief; Twenty-style relink events noted as follow-up).

## Routes (`server/server.mjs`, additive) + permission gate
- Flag: `FEATURE_TASKS` → `features.tasks` (the 2 pre-granted `env.mjs` lines). `parts[1]==="tasks" && !FEATURES.tasks → 404` beside the other flag gates; nav+page hiding is free (App.tsx `visiblePages` filters by page key).
- `/api/tasks`: GET list (filters `status` · `assignee` · `record=<obj>:<id>` · `due=overdue|today|week`; links enriched at READ time with live `label` (target's primary) + `objLabel`; response `{ tasks, me }`, `me` = session account name or null — feeds my-tasks-first) · POST · PATCH `/:id` · DELETE `/:id`. Route-side validation: title required; status enum; due `^\d{4}-\d{2}-\d{2}$` + real date; each link names a known object + a LIVE record.
- Gate (mirrors the apikeys role check, opened to members): `role` is already computed in `api()` (key role / team-derived / owner when auth off). GET = every role; POST/PATCH/DELETE deny `viewer` → `403 "your role (viewer) cannot manage tasks"`. No `can()`/ACTIONS change — tasks are a non-object surface (like teams/webhooks); client twin = the same `auth.role !== "viewer"` check hiding mutating affordances (server stays the real gate).
- Assignment mail (ONE block, POST + PATCH when assignee set/changed): resolve name → live account user; `sendMail(store, { to, kind: "task-assigned", subject: "Task assigned: <title>" , text: due + /#/p/tasks link })`; actor excluded (self-assignment silent). Route-side so replay never re-mails.

## UI
- `src/app/pages/Tasks.tsx` (new) + `{ key: "tasks", label: "Tasks" }` in pages.tsx. Composer: title `tasks-new-title`, native date `tasks-new-due`, assignee select over the users directory `tasks-new-assignee`, `tasks-create`. Filters: `tasks-filter-status` / `tasks-filter-assignee` / overdue checkbox `tasks-filter-overdue`. Buckets in order Overdue/Today/This week/Later/Done (`tasks-bucket-<key>`, empty hidden, no-due → Later). Rows: done toggle `task-done-<id>`, title, due, assignee, record chips `task-link-<tid>-<rid>` → `#/o/<obj>/r/<rid>`, delete `task-del-<id>`. Mine-first (assignee === me), then due-asc within buckets. ~10 new i18n keys (buckets + composer).
- Record block: STARTER-SIDE COMPOSITION (no RecordPage prop needed) — `RecordTasksBlock` exported from Tasks.tsx, rendered by RecordView in a fragment AFTER `<RecordPage/>`: full-width card below the record grid (frozen RecordPage has no slot — the accepted tradeoff). Open count `record-tasks-count`, rows with done toggle `rtask-done-<id>` + unlink `rtask-unlink-<id>` (PATCH links minus this record), inline add `rtask-title|due|assignee|add` posting `links:[{obj,id}]`. Refresh: existing `load()` + rev-poll (task `_ev` bumps the object rev). Viewer role hides mutating affordances. `src/app/api.ts` gains `tasks/taskCreate/taskPatch/taskDelete` + `TaskItem` type.
